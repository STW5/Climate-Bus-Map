## 1. 개요

기후동행카드 앱의 핵심 기능인 **기후동행카드 이용 가능 여부 판단 로직**에 버그가 있었으며, 공식 기준을 재확인하여 정확도를 개선했다.

## 2. 문제 분석

### 기존 코드 (`climateChecker.js`)

```js
if (/[가-힣]/.test(no)) return false; // ← 마을버스 잘못 제외
if (/^[MGA]/i.test(no)) return false;
if (/^N\d+$/i.test(no)) return true;
if (/^\d+$/.test(no)) return true;    // ← 9xxx 광역버스 잘못 포함
```

| 버그 | 내용 | 영향 |
|---|---|---|
| 마을버스 오제외 | 한글 포함 노선번호(`강남01` 등) → ❌ 처리 | 기후동행 가능 노선을 불가로 표시 |
| 9xxx 오포함 | `9000`, `9714` 등 광역버스 → ✅ 처리 | 기후동행 불가 노선을 가능으로 표시 |
| ODsay type 미사용 | `lane[].type` 코드가 있었으나 노선번호 패턴만 사용 | 더 정확한 데이터 활용 못 함 |

## 3. 공식 기준 확인

**조사 방법:** 서울시 공식 안내 페이지, ODsay API 공식 문서 직접 확인

**서울시 공식 기준 (news.seoul.go.kr):**
> "서울시 면허 시내·마을·심야버스 사용 가능, 광역·공항·타지역 면허 버스 제외"

**판단 원칙:**
1. 서울시 면허인지 확인
2. 광역/공항버스가 아닌지 확인
3. 두 조건 통과 시 기후동행 가능

---

## 4. 해결 방법

### ODsay `lane[].type` 코드 활용 (1차 판단)

ODsay `searchPubTransPathT` API는 버스 구간 `lane[].type`으로 버스 종류를 이미 반환하고 있었다.

| type | 종류 | 기후동행 |
|---|---|---|
| 3 | 마을버스 | ✅ |
| 11 | 간선버스 | ✅ |
| 13 | 순환버스 | ✅ |
| 1 | 일반(지선) | → fallback |
| 14 | 광역버스 | ❌ |
| 5 | 공항버스 | ❌ |
| 4 | 직행좌석(경기) | ❌ |
| 6 | 간선급행(M버스) | ❌ |
| 10 | 외곽 | ❌ |

### 노선번호 패턴 수정 (2차 fallback)

```js
if (/^[MGA]/i.test(no)) return false;  // M버스, 경기, 공항
if (/^9\d+$/.test(no)) return false;   // 광역 9xxx ← 추가
if (/^N\d+$/i.test(no)) return true;   // 심야
if (/[가-힣]/.test(no)) return true;   // 마을버스 ← false → true
if (/^\d+$/.test(no)) return true;     // 간선·지선·순환
```

### 최종 로직 구조

```
버스 구간 판단
  ↓
ODsay type 코드로 판단 (1차)
  ├─ INELIGIBLE(4,5,6,10,14) → ❌
  ├─ ELIGIBLE(3,11,13) → ✅
  └─ 불확실(1,2,15 등) → 노선번호 패턴으로 fallback (2차)
```

## 7. 한계 및 향후 개선

- ODsay type 1(일반), 2(좌석), 15(급행) 등 애매한 타입은 여전히 노선번호 패턴에 의존
- 완전한 정확도를 위해선 서울시 버스 면허 DB 조회가 필요하나, 공개 API 미제공으로 현재 불가
- 현 구현은 ODsay type + 패턴 조합으로 실용적인 최선의 정확도 달성
