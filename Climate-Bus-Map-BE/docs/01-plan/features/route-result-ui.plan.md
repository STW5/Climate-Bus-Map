# Plan: 경로탐색 결과 UI 개선

> 카카오맵 수준의 경로 결과 UI — 구간별 소요시간, 도보시간, 부드러운 애니메이션

- **작성일**: 2026-03-17
- **레벨**: Dynamic
- **상태**: 계획 중

---

## 1. 문제 정의

### 현재 문제

- 경로 결과에 총 소요시간만 표시됨 (구간별 시간 없음)
- 도보 구간 시간 미표시
- 카드 색감이 단조롭고 애니메이션이 딱딱함
- 교통수단 아이콘이 이모지 수준으로 조잡함

### 해결 목표

> 카카오맵처럼 각 구간별 소요시간과 도보 시간이 명확히 보이는 UI
> 부드러운 슬라이드 애니메이션과 세련된 색감 적용

---

## 2. ODsay 응답 활용 가능 데이터

```
path.info
  ├── totalTime       총 소요시간 (분)
  ├── totalWalk       총 도보 거리 (m)
  └── totalStationCount  총 정류장 수

path.subPath[]
  ├── trafficType     1=지하철, 2=버스, 3=도보
  ├── sectionTime     구간 소요시간 (분)
  ├── distance        구간 거리 (m)
  ├── stationCount    정류장 수
  └── lane[].busNo    버스 번호 / subwayCode 호선
```

---

## 3. 핵심 기능

### F-01: 구간별 소요시간 표시

- 각 subPath의 `sectionTime` 표시
- 도보(trafficType=3): "도보 N분" 표시
- 버스/지하철: "N분 (M정류장)" 표시

### F-02: 경로 요약 바

- 카카오맵 스타일 구간 비율 바
- 각 구간을 색상으로 구분 (지하철=파랑, 버스=초록, 도보=회색)
- 기후동행 불가 구간은 빨간색

### F-03: UI 색감 및 애니메이션 개선

- 결과 패널: 아래서 슬라이드업 애니메이션 (0.3s ease)
- 카드 hover: 부드러운 scale + shadow 변화
- 선택된 카드: 왼쪽 액센트 바 + 배경색 변화
- 색상: 기존 단색 → 그라디언트 + 소프트 팔레트

### F-04: 정류장 수 / 도보 거리 보조 정보

- 버스 구간: "N정류장" 표시
- 도보 구간: "약 Nm" 또는 "N분" 표시
- 총 요약: 소요시간 + 총 도보거리
